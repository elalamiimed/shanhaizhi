# 《山海经——人生旅行相册》 (Shanhaijing)

> 一款结合国风《山海经》神话美学与真实地理的旅行手帐打卡游戏。

## 🎮 游戏简介

上古时期，山海界与现世重叠。后绝地天通，神话隐没于名山大川之中。玩家扮演现代的"巡界司"传人，用真实的脚步和感悟，重新为山海界注入色彩，唤醒沉睡的神明与异兽。

面对一张初始灰色的华夏卷轴地图，玩家可以**选择曾去过的城市，写下专属的旅行记忆并打卡**。每一次打卡都会点亮该城市，收集对应的山海经神兽卡牌，将地图从"太虚混沌"变为"神话金"。游戏通过浏览器本地存储（LocalStorage）记录玩家的每一步足迹与文字感悟，打造独一无二的人生旅行手帐。

## 📍 在地图中珍藏记忆 (Saving Memories in the Map)

每一次真实的旅行，都是一次与山海界的共鸣。
- **足迹同步**：游戏内置 `china-memory-map.html` 交互式地图，与主引擎通过 `shanhai_travel_save` 共享记忆数据。
- **点亮神州**：打卡后，城市将变为专属的中国传统色（如东山经的朱砂红、西山经的敦煌黄）。
- **记忆封存**：玩家写下的旅行感悟将被永久保存在对应的地理坐标上。随时点击地图上的城市节点，即可重温当时的旅行记忆，并查看该地解锁的神兽引文（如九尾狐、白泽等）。

## 🎵 抖音生态接入方案 (Implementation in Douyin)

本项目目前为纯前端单机运行（基于文字记录的轻量版）。若将其作为**抖音小程序（Douyin Mini Program）**进行深度定制，可依托抖音强大的LBS（基于位置服务）和多媒体生态，实现以下进阶玩法：

1. **一键授权与地理位置打卡 (POI 联动)**
   - 告别手动选择城市。当用户在抖音发布带有特定地点定位的视频或图文时，小程序自动识别并点亮游戏内的对应城市。

2. **视频/图文内容沉淀为地图记忆**
   - 突破目前的纯文本感悟限制。玩家在抖音发布的旅行 Vlog 和实拍照片可以直接挂载到游戏地图的对应城市节点上。地图不仅保存文字，更成为用户的"全景视频旅行手账"。

3. **云端记忆同步**
   - 突破纯前端 LocalStorage 的单机限制，利用抖音小程序的云开发（Cloud Development）或用户账号体系，实现跨设备、永不丢失的云端记忆存储。

## 🔗 好友分享与社交互动 (Sharing with Friends)

旅行的意义在于分享。在抖音生态下，游戏的社交属性将被无限放大：

- **神兽卡牌与海报分享**：玩家解锁稀有神兽卡牌后，可一键生成带有专属旅行数据（如"已点亮全国34%的版图"）的精美水墨风海报，直接分享至抖音日常或朋友圈。
- **好友邀请与助力**：
  - **组队寻妖**：邀请抖音好友共同点亮某个大区（如"东山经"），完成后双方皆可获得稀有成就。
  - **异地互踩**：好友之间可以互相访问对方的"山海地图"，查看TA的旅行足迹与记忆感悟，并在TA的地图上留言或"沾沾仙气"。
- **抖音视频锚点 (挂载组件)**：在发布的旅行视频左下角挂载小程序链接，观众点击即可看到创作者当前的"山海经点亮进度"，并一键开启自己的神话旅行。

## 🚀 快速开始 (本地开发)

纯前端单机运行，无需安装任何依赖：
```bash
# 克隆或下载项目
cd shanhaijing-travel

# 使用 Python 启动本地服务
python3 -m http.server 8080

# 或使用 Node.js
npx serve .
```
访问 `http://localhost:8080` 即可体验。

## 📁 核心技术说明

- `engine.js`: 核心引擎，负责数据存储、状态管理（打卡验证、神兽解锁等）及子系统调度。
- `ui.js`: 负责页面渲染、手动打卡交互（城市选择与感悟输入）及弹窗逻辑。
- `china-memory-map.html`: 基于 Leaflet 的交互式地图 iframe，支持双向数据同步机制（通过读取 `window.parent.GameEngine._state` 或 fallback 到 `localStorage`）。
- **纯本地存储**: 保护用户隐私，所有打卡记录与文字记忆均存储在浏览器本地（适配抖音小程序后可平滑迁移至云数据库）。
